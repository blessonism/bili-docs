## 研发 TimeGPT 的初衷

Hello 大家好，今天来给大家录一个 TimeGPT 方法的视频。我把之前50多分钟的视频（可能之前有人看过）中一些重要的地方提炼出来，以图形化的形式更简单地给大家讲解一下。我想要做这个东西是在2024年的时候，那时候我刚刚毕业不久，在想想要去做什么事情。那个时候有非常多的选择，有非常多的事情可以做，所以比较迷茫。那个时候我就自己去做了一些小小的研究。

## 传统四象限法则的局限性

最开始我发现，我其实想要给我的这些事情排一个优先级，我应该先做哪一些事情。那个时候我就发现了艾森豪威尔矩阵，也就是大家常说的紧急重要性矩阵。看我视频的观众大部分应该都知道了，很简单就是两个维度：紧急性和重要性。对于每个任务来说，我们最想要去做的事情就是又紧急又重要的事情。

但是这个东西我认为有两个问题。第一个是，对于同处于紧急和重要象限里的任务，比如说现在有 Task 1、Task 2 这两个任务在这里，我没有办法去比较应该先做哪一个，这是第一个问题。第二个问题是，紧急性对于我们来说是“我要不要去开始这个任务”，而重要性对于我们来说是“做完这个任务之后的结果是什么样”。所以我感觉它是忽略了中间过程的部分。

## 任务评分系统的五个维度

举个例子，假如说现在同时有两个任务，第一个任务的紧急性是4分，第二个任务是5分；第一个任务的重要性是10分，第二个任务的重要性是9分。如果第一个任务的花费非常少，第二个任务的花费非常多，那么我们其实能够非常轻易地选出，应该先做花费比较少的任务。

对于刚刚这两个问题的解决方法，第一是我们把单纯的象限划分方式变成一个评分系统。比如说1号任务的重要性是2，紧急性是4；2号任务的重要性是4，紧急性是3。简单一点来说，就是把这两个数字相加，哪一个的分数更高，我就应该先做哪一个。第二是因为我觉得它缺少了过程的维度，所以我们加上过程的维度。

但是这三个东西目前都只针对于任务本身。如果想要往长远去思考，我其实想要去看这三个维度背后的项目，以及项目背后的目标等不同维度。如果我们把目标分成项目，项目分成任务，长远来说我们想要去完成目标，短期来说我们是完成任务。那么重要性和花费就能再分成两个不同的维度。对于短期来说，我单独看任务本身的花费；长远来看，我去看这个任务背后的目标所需要花费的总努力是多少。对于重要性来说，短期是看任务的价值，长期是看这个目标背后的影响力是什么样。

这样的话，我们要完成一个完整的目标，它的开始、过程以及结果，一共就得到了五个维度：紧急性、花费、努力、影响以及价值。

## 引入层次分析法（AHP）计算权重

有了这五个维度之后，我们对于每一个任务都去评这五个维度的分数，然后得到每一个任务的总评分，这就是把它变成一个评分系统的方式。但是对于这五个不同的维度，每一个人的权重是不一样的。在上学的人可能觉得时间不是那么重要，更看重任务背后的影响力以及任务本身的价值；同时学校里作业一般都有截止日期，所以更看重任务的紧急性。对于初入社会的人来说，可能觉得时间相较来说更重要一些，紧急性没那么重要了，需要考虑怎样花费更少的时间获得更大的结果。

对于这五个不同的维度，每个人心中都有一个权重。当然我们可以简单地给它们标一个数字来分配权重，但这样有一个问题，就是你没有办法去比较。单纯觉得紧急度打1到10分是多少，另一个维度打1到10分是多少，它们之间没有相互比较的过程，这样意义不大。

为了解决这个问题，我们引入了层次分析法。举个例子，如果我现在想要配一台新的电脑，可能要考虑电脑的三个维度。因为预算有限，不能每一个地方都买最好的，所以我就想这5000块钱应该怎么分配在 CPU、GPU 以及主板上。这个方法其实就是让它们两两之间进行比较。自己和自己比较都是1。比如 GPU 和 CPU 比较，我认为 GPU 更重要（比如我想打游戏），所以 GPU 和 CPU 比的时候是2，相应的 CPU 和 GPU 比的时候就是1/2。其他也是一样的，我觉得 GPU 和主板比较时 GPU 是3，相应的主板和 GPU 比较就是1/3。CPU 和主板我觉得差不多。

最终就能得到这样一个表格。对于每一列来说，我们可以得到一个权重，按百分比比例计算。再把这行加总在一起，我们就得到了 CPU、GPU 以及主板所需要的权重。最终，任务的总分数就是每一个维度的分数乘以它的权重，然后加总得到的分数。

## 基于线性规划的任务推荐系统

有了这个分数之后，我其实还想在系统里做一个推荐系统。比如我有10个任务，我想要系统每天给我推荐其中的几个任务。依据什么条件来推荐呢？这里其实用到了一个线性规划的过程。我想要去限制它的规则有：第一，每天有一个固定的时限，选出任务的加总时间不能超过某个时间，比如不能超过5个小时或6个小时；第二，想要最大化选出来的这几个任务的分数；第三，任务会有不同的类别，我希望依照类别给任务做一些额外的分配。最终我们就通过线性规划去找到了推荐的任务组合，计算出它们的分数是多少，以及有多少时间花费。

## 任务的三种分类：杂事、事件与任务

刚刚讲到任务类别，现在一共分为三种。我们可以通过一个流程图去判断它是哪一种。首先一个待办进来，看它有没有一个大的目标。如果没有，它其实就是一个日常生活杂事的提醒，比如需要回邮件、倒一下猫砂、买菜等等，这就属于“杂事”类别。

如果它是有目标的，说明这些事情可能是比较重要的事情。接着看它有没有一个起始日期，有没有一个具体的开始时间。如果有，说明它是一个“事件”；如果没有，那就是一个普通的“任务”。事件和任务的区别在于，事件可能是日常生活中的会议，或者跟医生预约的时间，只有到那一天的时候才要去做，不能提前也不能延后。而任务是可以自由安排什么时候去做的，只要在预定的截止日期之前把它做完就好了。这就是三种不同的任务类型。

## 时间管理与间歇日记

刚刚讲的都是关于任务管理的部分，接下来是时间管理的部分。在时间管理里，我们采用的是间歇日记的方式来进行记录。间歇日记其实很简单，就是一个时间戳加上你做了什么事情。一天记录下来之后，时间戳和时间戳之间就有时间段，你可以对每一个时间段进行分类，得到这个时间段的时长。每天你就能够汇总在不同的时间分类上花费了多少时间。这个方式说实话也没有比其他方式好非常多，但这是我过去一年到现在为止能够坚持下来的方式。

接下来是时间的分类。以我目前的经验来说，这六个不同的类别应该能够涵盖大家生活中常见的大部分时间类别了：第一是主要工作，学生的主要工作是学习，上班的人当然就是工作；第二是自我提升，自己去看书、阅读或者学习新技能的时间；第三是休息；第四是人际关系，和伴侣、朋友、家人相处的时间；第五是健康，比如去看医生、健身、运动的时间；第六是日常生活，就是我们所谓的琐事，比如洗衣服、做饭等等。

## 精力管理与精力热度图

虽然有了这些时间的分类，但我们大概也能知道在做哪些任务时是需要高能量的。比如去学习、阅读的时候，需要较高的能量；做其他事情的时候，不太需要高能量。我们就能给这些时间分类去定义它们是高能量还是低能量。

前面我们通过间歇日记记录了一整天的时间分配和分类，就可以得出一天中什么时间做了高能量的事情，什么时间做了低能量的事情。我们把一个小时分成12份，五分钟一份。就能得出这一个小时里，比如前20分钟做的是低能量的事情，后40分钟做的是高能量的事情。如果每一个小时都记录下来，就能得到一整天的精力热度图。如果每天都记录，最终就可以得到一个平均的精力热度图。也就是说，我这个人平均下来在什么时间更愿意去做高能量的事情，什么时间更愿意去休息或做低能量的事情，这是真实的数据反映。

## 任务、时间与精力的系统闭环

最终结合前面的任务推荐系统，我们去找在哪些高能量的时间段，它的时长和任务所需的时间是匹配的。因为高分任务优先级最高，所以我先去分配它，把它安排到精力最高的时间段，然后其他任务也继续依次分配。这样不仅能推荐每天的任务，还能根据我的精力去分配什么时间最适合做这些任务。

到现在为止，我们就完成了一个闭环：任务管理、时间管理以及精力管理。每天用日记的形式记录真实的时间发生情况，得到自己的精力热度图。有了精力热度图之后，能够更好地分配任务。而任务反过来又帮助我们更好地管理时间，去做更高效、更有用的任务。这就是一个非常好的循环。
