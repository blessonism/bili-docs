## 团队都在用的终端：Ghostty

Claude Code 负责人 Boris 说，他的团队最爱的终端就是 Ghostty。

我之前一直用 iTerm2，觉得够用了。直到我刷到 Boris 的推文，他说他同时开五个实例并行工作，全靠 Ghostty 的分屏管着。我就想，Claude Code 的编程设置，肯定是 Web Coding 效率最高的方式，跟着他学肯定没错。

## 一键配置：让 Claude Code 按配置把 Ghostty 配好

安装直接去 Ghostty 的官网下载就行。装好之后打开 Claude Code，跟它说：

帮我按照这个 config 内容配置 Ghostty。

然后只要把拷贝链接复制进去，它就会开始下载并配置整个过程，一键配置完，很简单，一目了然。不像 iTerm2 里面巨大的设置菜单，进去以后人直接麻了。

我把我的配置已经贴到 GitHub 上了，大家有兴趣可以直接使用。

## 主题：几百个配色方案实时预览

最让我上头的第一个特点是视觉主题。

比如在终端里输入 `ghostty +list-themes`，你会看到实时预览几百个配色方案。看到哪个喜欢的，直接记住它的名字，然后让 Claude Code 帮你设置好这个主题。

## Quick Terminal：随叫随到的下拉终端

还有一个特别好用的功能叫 Quick Terminal。按一下 Ctrl + `，屏幕唰的一下就滑下来；用完再按一下就收回去了。

比如有时候你突然要跑个命令，不用切来切去，直接呼出来，敲完就走。比如 `cat fi`，特别方便。

## 分屏 + Tab 工作流：写代码、看文件、看改动一套打完

配好了就要干活了。我用得最多的就是分屏管理。

我的工作习惯一般是左边开一个 Claude Code，右边按 Command + D 左右分屏。然后这个屏幕我用 yazi 浏览文件，比如这个项目里哪些文件结构需要的；还有可能看看图片素材、PDF 文件之类的。

接下来我会在下面用 Command + Shift + D 再开一个分屏，这里面我会用 lazygit 查看一下刚 Claude 的输出结果，看一看具体改的是一些什么东西。

有的时候你可能会多个项目同时工作，这时候我会用 Command + T 打开一个新的 tab，然后重复刚才的工作流：打开 Claude Code、yazi、lazygit 这些；再用 Command + Shift + 左右括号，在不同 tab 之间来回切换。工作完成以后，按 Command + W 关闭这个 tab 就行。

这部分和 iTerm2 是完全一样的，上手非常自然。

## 总结：Ghostty 打动我的三点

总结一下，Ghostty 打动我的就三点：

1. Claude Code 一句话配好，不用折腾。
2. 几百个主题随便换，看到想截图发朋友圈。
3. 分屏操作丝滑流畅，体验 GPU 渲染的流畅感。
