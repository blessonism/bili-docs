更加有前途
要用MCP的方式获取FIGMA的设计资源
首先你要将你的cursor
更新到最新的0.46版本
然后你需要获取一个访问FIGMA原数据的API key
这个API key可以在FIGMA的账号设置中免费获取
接下来你就要搭建一个FIGMAMCP的服务器了
你可以参考一下这个FIGMAMCP的GITHUB页面
他介绍了两种方式
我比较建议的是采用最简单的方式
你只需要在你的终端中执行这个命令
记得将你刚刚申请的figment API key
替换上去就可以了
当你看到终端显示了new SSD connection established
就说明FIGMA服务器已经搭建好了
这时你需要把s s e connection的链接复制下来
然后在cursor的设置中打开MCP的标签
创建一个新的m cp server
填上名字
选择SSE
将刚刚复制的链接填上就可以了
这样你的cursor就连接上了FIGMA的MCP服务器
接下来来到FIGMA里面
找到你要开发的设计页面
点右键选择copy link to selection
这样你就复制好了你的FIGMA文件的链接
回到cursor里面
确保你选择了agent的模式
以及cloud的任意模型都可以
然后让cursor根据FIGMA的链接来实现前端页面
在agent执行任务的过程中
会尝试去调用MCP服务器提供的工具
来获取设计图的原数据
这时你需要手动点击run tool来执行
否则cursor就会一直停止不动
在这个过程中
你需要多次手动的接受他的请求
来确保它能顺利完成所有必要的任务
值得注意的是
在这个过程中
cursor对于MCP服务器的连接可能会不稳定
这样会导致很多数据获取失败
