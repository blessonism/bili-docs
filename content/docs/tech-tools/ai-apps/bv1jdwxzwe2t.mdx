## 开场：今天给 Claude Code 20 个命令评级

作为 Claude 重度用户，今天我们来给 Claude 20 个命令评级，从“人上人”到“拉”，看看哪些是隐藏款真神，哪些是凑数的工业垃圾。

评分标准：  
“夯”就是没它我活不了；“拉完了”就是纯纯的电子垃圾。

## compact：压缩对话历史省 token

compact 用来压缩对话历史省 token，但关键信息全保留。对话太长 Claude 开始失忆，但你又不想丢掉上下文的时候，这就是救命稻草，长对话必备。这里给到一个“夯”。

## exit：结束会话的“仪式感”

exit 退出 Claude Code，结束当前会话，比直接关窗口优雅一点，但也就那么一点。适合有强迫症的程序员，总觉得得有个正式的结束仪式。

但这我真的需要告诉你吗？我直接 Ctrl+D 秒了。谢谢这个命令的存在，让我严重怀疑产品经理是不是在凑 KPI，纯纯占位置。这里给到一个“拉完了”。

## agent：启动子代理，多线程干活

agent 启动子代理，让 Claude 分身干活：一个查代码，一个写测试，大项目重构，多文件并行操作，实用效率直接起飞，而且上下文互不影响。

但需要很多脑力做好任务拆分就是了。这里给到一个“夯”。

## help：查帮助文档，新手村 NPC

help 查帮助文档。第一次用的时候瞅一眼，或者忘了命令叫啥的时候翻一翻，功能还算全面，基本上所有的斜杠命令都能在这里找到，简要说明算是新手入门导师。

但就像游戏里的 NPC，新手村聊两句，后面谁还找他，用完几次之后基本就遗忘了，存在感比空气还稀薄。这里给到一个“NPC”。

## diff：看每轮改动，防止 AI 搞事

diff 想知道 AI 是在帮你还是在搞，你改了哪些文件、每轮对话都动了什么，这个很实用。

按左右键切换当前所有改动和每轮对话的改动，按上下键浏览不同文件。程序员可以看代码，也可以看 Markdown 里面的修改，防止 AI 误解我们的意思，先下手为强。这里给到“人上人”。

## hooks：配置钩子，自动化工作流拉满

hooks 配置钩子，在特定事件时自动执行命令，比如每次提交前跑测试。想自动化工作流、强制执行规范的时候用。

配好了之后，整个开发流程丝滑得不像话，配置一次爽一年。但配置过程会让你怀疑人生，各种配置文件看得人头大。不过配完了是真的香，自动化程度直接拉满。这里给到一个“顶级”。

## clear：清空上下文，重新开始

clear 清空上下文重新开始。当 Claude 开始胡说八道、陷入死循环，或者你想换个话题使用，特别是对话讲久了 Claude 注意力会分散，这时候清一下，用完感觉像换了一个新的 Claude，之前的各种混乱逻辑瞬间归零，简直不要太爽。这里给到一个“夯”。

## /copy：复制对话到剪贴板

/copy 把对话内容复制到剪贴板。想把 Claude 的回答贴到别的地方用，比如写文档或者跟同事对线，特别是 Claude 给你写了一大段解释，直接复制去邮件里发给老板，省得自己再整理一遍。

有时候就在想直接鼠标选中复制不行吗？但用了几次之后发现这个命令能复制整个对话的格式化版本，也算有点用。这里给到一个“NPC”。

## mcp：管理 MCP 服务器，连接外部工具

mcp 管理 MCP 服务器，让 Claude 连接外部工具和数据库。要用 Claude 操作数据库、调 API、接外部服务时就得配置。

搞明白了之后，Claude 瞬间从普通 AI 变身超级助理，各种外部服务手到擒来，专业玩家区域。但刚接触的时候一脸懵，什么服务器配置、协议对接，听着就劝退。

主要问题是过于花费 token，建议有心理准备再入坑。这里给到一个“顶级”。

## context：查看当前上下文都有什么

context 查看当前上下文里都有啥，Claude 记住了哪些文件。有时候加了太多文件到上下文，Claude 会选择性失忆，用这个检查一下它到底记住了什么。

偷看 Claude 的记事本，看它有没有在听你说话。有时候发现它根本没记住关键文件，就像那种表面点头其实走神的学生，被你当场抓包。这里给到“人上人”。

## plan：先写方案再动手

plan 进入规划模式，让 Claude 先写方案再动手。复杂任务不确定怎么实现，或者怕乱改的时候，这就是“公投模式”，先画蓝图再搬砖。

真正懂工程的人才用这个避免返工，省下的时间比你想象的还多。这里给到一个“夯”。

## dr：诊断 Claude Code 环境

dr 诊断 Claude Code 环境，检查配置有没有问题。出 bug 了、连不上 API 了、行为异常了，先跑个 dr，基本上 90% 的问题它都能帮你找到原因。

就像电脑重启能解决 90% 的问题，另外 10% 解决不了的问题，可能需要你真的重启电脑了。不过这个诊断报告还算详细，不至于让你抓瞎。这里给到一个“NPC”。

## skills：管理自定义技能，相当于装插件

skills 管理自定义技能。你可以给 Claude 装各种扩展能力，有重复性、知识性工作想自动化时，自己写个 skill 让它记住，相当于给 Claude 装插件。

比如让它记住你的代码风格或者固定工作流程，越用越顺手，效果很好用。而且使用了渐进式披露，上下文占用很少，你可以训练它做很多具体任务，性价比之王。这里给到一个“夯爆了”。

## init：初始化项目，生成 CLAUDE.md

init 初始化项目，生成 CLAUDE.md 配置文件。新项目第一天用，让 Claude 知道你的项目规矩，相当于给 Claude 发一本员工手册，告诉它公司的规章制度和技术栈，以后合作起来更顺畅。

第一次用 Claude Code 必跑，后面每次打开一个新窗口都要用一遍，不然 CLAUDE.md 没几天就开始变成捣乱分子了。这里给到“顶级”。

## btw：顺便补充背景信息，不打断对话

btw 也叫“顺便说一下”，可以在不打断 Claude 的前提下给它补充一些背景信息，下次不用重复。

比如 btw：我用的是 TypeScript，那么这轮对话它就不会给你写 JS 了。发现有什么东西不小心忘说了或者说错了的人类程序员，同时又不想再开一个窗口重置上下文、浪费 token，by the way 还是很高频的一个指令。这里给到一个“顶级”。

## cost：看这轮对话花了多少钱

cost 看当前对话花了多少钱，token 消耗明细。老板问你 AI 花了多少银子的时候，或者你自己手贱好奇的时候，用它数据还算详细，能看到每次对话的成本，方便做预算控制。

但是别每次调用都看，看完你焦虑，就像没事盯着股票账户一样。建议每周看一次，在心理承受范围内。这里给到一个“NPC”。

## insights：代码洞察分析，考古神器

insights 代码洞察分析，帮你理解代码结构和潜在问题。接手史山项目或者想知道这代码到底在干嘛，用它能快速理清项目的架构逻辑，找出潜在的性能瓶颈和设计缺陷，也可以偶尔用来了解自己的编程习惯。

考古学家的洛阳铲，挖别人代码必备。特别是接手那种没有任何文档的老项目，这就是救命稻草，能帮你少走很多弯路。这里给到“人上人”。

## loop：循环任务调度，自动巡航

loop 循环任务调度命令，定时自动执行、监控部署状态、定期检查，自动化流程实用，相当于给 Claude 装上了一个自动巡航系统，用起来有点 OpenAI 那个味。

唯一的缺点是你得确保任务配对了，不然 Claude 会定时帮你犯错，而且还是批量犯错，那就尴尬了。这里给到一个“顶级”。
