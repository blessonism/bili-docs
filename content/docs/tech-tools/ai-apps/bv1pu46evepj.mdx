## 开场：今天分享什么

每天分享一个 AI 在各行业的应用案例。大家好，这里是 AIGC Link。今天分享一个 5 分钟可以上线企业级 AI 应用的平台，它比 Coze、Dify 更强大。

## Zion 平台的核心特点

它是首个将 AI Agent（智能体）和前端、后端、数据库完美整合的 AI 应用构建平台。通过它可以实现 AI Agent 的应用开发，并且能够发布到小程序、网站和 SaaS 服务等各端，还可以收费。

发布网站或者 SaaS 服务的时候，可以直接设置收费服务、用户登录和会员管理。整个过程不需要写代码。

## 产品定位：像“Coze + 网站构建平台”的融合

我认为它有点像原先的一些网站构建平台，融合了 Coze 的能力，形成“智能体 + 网站”的一体化平台。

## 案例展示：AI 故事插图应用怎么做的

这是我们用它构建的一个“AI 故事插图”应用，用到的能力包括：

- AI 生成故事  
- AI 根据故事做分镜  
- 再把分镜提示词交给模型（例如 DALL·E 3）生成对应插图的提示词  
- 最后生成插图

## 为什么说它是生产级：自带数据库与完整应用能力

这个应用是生产级的，可以直接发布，因为它有数据库。比如能看到：

- 账户数据库  
- 故事数据库  
- 分镜数据库  
- 会话消息等各种数据库

它不是那种只有前后端展示、没有任何记录存储的简单页面。

预览页面里你可以直接对话，比如让它生成一个“小孩洗手”的故事：它会把提示词交给大模型，调用“生成故事”的 Agent（示例里用的是 GPT-4o/类似模型），输出 title 和 content。

生成插图时，它会先做分镜：每个分镜有序号、标题和描述。你也可以根据情况自定义修改，然后再生成插图。生成图片需要时间，我这里就不等了。

## 一键发布：域名、小程序、SaaS、收费与登录

这个应用可以发布到自己的域名上，也可以用官方免费的域名。自定义域名的话，配置 DNS 的 CNAME 就能直接访问。

发布后就是一个完整应用，包含登录、注册、数据库存储等能力。这是它区别于其他平台的地方。

## 从零实操：新建项目（网页/小程序）

下面实操从零构建一个 AI 应用。

新建项目时有两种入口：

- 小程序
- 网页

我这里从网页开始。创建后会自动有几个页面规格：桌面端、手机端、Pad 端。

## 构建第一个 Agent：AI 方案助手

先构建一个 Agent，我用 3.5 模型演示。

- 入参：question（文本）
- 任务角色：AI 助手
- 任务详情：根据用户提问生成对应行业解决方案
- 任务限制：不提供与用户提问无关的话题

它还支持知识库，也支持数据库：可以接平台内建数据库，也可以通过 API 调用公司自己的数据库/API。API 库配置在 API 模块里。

输出我设置为两个字段：

- title（方案标题）
- content（方案内容）

发布后可以直接对话测试，比如问教育行业的解决方案，它会返回对应内容。也可以问知识库里的问题，例如“开发者大会开会地点在哪里”，如果知识库 PDF 里有，就能回答出来。

## Workflow / API / 多 Agent：更复杂的编排能力

除了单 Agent，还可以构建 workflow：

- 同步：从上到下顺序执行
- 异步：多个节点同时执行

workflow 里可以引入已有 Agent，把页面数据或入参数据映射到节点输入。

API 也可以自定义配置（header 等），然后在 Agent 或 workflow 里调用第三方 API。

另外它支持多 Agent：Agent 里面再调用 Agent，用来构建“团队式”能力，这点比较强。

## 重点演示：页面 + 数据库 + Agent 串起来

重点给大家看：用前端页面和数据库去关联后面的 Agent。

### 页面搭建：输入框 + 提交按钮 + 条件容器

我先加一个“条件容器”（后面用来切换结果视图），再拖入：

- 输入框
- 提交按钮

在按钮上设置行为：点击后调用前面创建的 Agent。

- question 参数：绑定到输入框组件的数据

### AI 执行成功后：写入数据库

AI 执行成功后，我希望把结果存到数据库，所以先建数据库表，比如“方案表”，增加字段：

- title（文本）
- content（文本）

然后在“成功时行为”里增加数据库写入：

- title = AI 结果数据里的 title
- content = AI 结果数据里的 content

### 切换到结果页：用条件容器展示输出

我还希望展示结果，所以需要切换视图到“结果页”。

这里要注意：我一开始把“跳转”设置错了。因为这是同一个页面内的视图切换，应该用“容器切换视图”到结果页；“页面跳转”一般用于登录注册等跨页面跳转。

### 页面数据绑定：把 title/content 渲染出来

为了在结果页展示，我创建页面数据：

- title（文本）
- content（文本）

然后在按钮行为里加“设置页面数据”：

- title = AI 结果数据.title
- content = AI 结果数据.content

结果页组件绑定页面数据即可展示内容。

发布预览后，输入“教育行业 AI 应用有哪些”，可以正常生成并切换到结果页展示。数据库里也能看到新增的方案记录。

我后面做了一点小迭代：把单行展示换成多行的富文本组件，用来更完整显示 content，并把默认的 “hello world” 去掉。

## 登录注册：可直接加账户体系

如果你想增加用户账户体系，可以直接加登录注册页面。提交按钮的行为里就能配置“用户事件 → 登录”。

做法是：

- 放两个输入框（用户名/密码或其他）
- 登录按钮点击行为选择“用户名登录”
- 用户名、密码分别绑定输入框数据

登录后的用户数据会出现在账户数据表里。登录注册在配置里打开开关即可，还支持手机号等方式。

## 支付、SEO、自定义域名等发布能力

它也支持支付、邮箱相关配置、自定义域名、SEO 设置等。升级后用自定义域名发布会更方便。
