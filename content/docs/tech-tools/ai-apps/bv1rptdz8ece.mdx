大家好，我是春秋。好久不见，这次我来给大家分享一下 Claude Code 的终极玩法。我已经用了差不多有三个月了，目前的感受就是，Cursor 就是个智障，所有的模型写代码都没有这个强。但是这东西在老外那边比较火，国内没什么动静，可能因为各种自媒体的水平有限，他们菜得要死，就只会天天搞那些东西，没什么真本事，不知道这东西牛逼在哪，也介绍不清楚。估计我一发出来，他们就开始跟着炒这个东西了。

我先带头讲一下，大部分人都是通过这个来认识我的。昨天有人来催更，我想那就做个视频吧，要不然大家都不知道我是不是还活着。我最近其实一直在写代码，就没怎么更了，也没什么热点事件。你说 Cursor 满大街都是介绍的，我有什么好介绍的，就懒得做视频了。所以别人玩不明白的，咱来搞一下。

## 终端配置与提示词优化

第一步，因为这是一个老外的东西，它不是中文的，要通过命令行操作。我提前给大家操作了一下，可以先看一下。先运行进到终端里面，去 memory 选最后这个用户的记忆，第一句写“用中文回答我”。后面是因为有时候你自己说话不是完全对的，你需要 Claude 来指出你的问题，其实它很多时候是比你强的。所以就让它“用审视的目光，仔细查看你输入的潜在问题”。反正这么写效果是很好的，可以优化一下效果。

## 命令行输入图片技巧

接着往后看，大家不知道命令行怎么输入图片。其实正常复制是 Command+V 粘贴，但是这里边用的是 Ctrl+V。我用的是 Mac，Windows 同学自己摸索一下。

## 结合 MCP 实现自动化测试与开发

接下来这个是最牛逼的，就是用 MCP。MCP 大家都知道，可以操作外部服务。结合这个你就可以自动做网站了，因为做网站每次都要把效果截图传回去。我做了一个比较简单的 Demo，大家可以看一下。

首先添加 Puppeteer 的 MCP，添加完之后进到测试的项目目录里面，输入 `claude` 启动，你看 MCP 就 connected 了。我打了一个提示词，说“帮我完成一个点击屏幕就增加数字”，很简单的，各种教学网站的例子都是用这个，咱主要看一下它用这个测试。它说“我来弄”，这东西牛逼就牛逼在它会先列一个 To-Do，然后一条一条划掉。

现在表示它正在做第一个 To-Do，在写 HTML 了，我这部分是没有加速的。这个时候大家可能有些人就点第一个了，不要点第一个，直接选第二个，让它一直自动写就好了，要不然总会卡住。你看这有个 auto accept（自动接收），自动接收它写的代码，反正写错了也没关系，勤用 Git 保存一下就行。然后它写了一个测试代码，开始测试了，问要不要弄，也是自动给它权限就好了。这是它自动开的，不是我开的，这是它自己写的，大模型都知道。

接下来是我点的，我给它切开，它又让我给运行权限，我就给权限了。你看，这是它自己进到这个路由的，它自己刷新。因为我前面点了两下，是个 2，结果它进过来之后就变成 0 了，它刷新了，这是没有问题的。接下来我移动了一下，没进行别的操作，它又要权限，你看它要一个 click 的权限，点的是标签里面的东西，它可以自动点。自动点完之后，它会用截图自动把这一块截掉，看一下点得对不对。你看它说“卧槽，数字从 0 变成了 1，让我们来多点几次”，然后运行第一行点一下，一共三次四次，没有问题，咔咔点三下。它又开始分析了，又截图分析看是不是对的，发现弄对了，结果把这个 To-Do 划掉了。

这就是厉害的地方。我这里只是举了一个比较简单的例子，真实的项目你想测一个模块、测一个插件、测一个控件，都可以这么做，只要装上了 MCP，前提是你得在本地。

## 数据库操作与自动更新文档

MCP 牛逼的地方不只是这个，我这里只是拿这个举了个例子。如果你用 Supabase 或者微服务的那些东西，它可以支持 PostgreSQL，能自动帮你创建表格、获取数据结构、修改表。数据库的所有操作你都不需要管了，它都能帮你一气儿写好，写完之后再帮你创建模拟数据。创建完之后它还会写文档、弄 Git，反正可智能了。最后它整个就知道了，无论前端接数据库还是后端接数据库，它都能接对。这就是这东西牛逼的地方，它能自动更新文档。

MCP 这个功能讲明白了对吧？可能大家还觉得，这不就是 MCP 吗，我 Cursor 也有。接着往后看，你看它会更新这个文档，Cursor 没有了吧？Cursor 确实可以用初始化来分析代码，但它不会把这个东西弄成一个自动更新的文档 `claude.md`。反正它会自动搞个这么个文件然后更新。

## 智能代码库分析与潜在问题排查

你看它就开始分析你的代码库了，开始一个一个查。而且它不是挨个顺序扫，它是先搜索、分类查。先看配置文件，然后再看别的。我这个项目太简单了，它后面分析得很智能，它是会调整的。它不像 Cursor 跟个呆瓜似的，只会一层一层地扫，它不是那样的，这个比 Cursor 强太多了。

你看这部分，因为我用了提示词，它给我了一些建议。这视频不能快进，我做视频水平有限，不会那些花里胡哨的剪辑，这已经是尽最大努力展示了。我每一段都录了视频，让兄弟们看清楚体会一下这东西的牛逼。

出来了，你看这就是它发现的潜在问题。它说这个东西不完整，`readme.md` 是空的，这是一个很不好的习惯。你看混合技术栈，因为刚才写测试用的是纯 HTML，就不太行，真正做项目没有这么做的，哪有 Python 和纯前端这么搞的。这些就是它发现的问题，你没讲这些，只是让它生成一个东西，它就发现了，这很牛。然后它会给你一个建议，让我学什么，怎么干。这个建议很中肯，这就解决了如果你技术不强，没办法很好指挥它的问题。它现在在教你，让你去考虑它的建议。如果没有那个提示词，它是不会输出这些东西的。

## 命令行快捷指令与无头模式

接下来是一个比较好的小技巧。如果是一些小事，我比较喜欢直接输入 `claude` 后面接一个字符串语句，接个命令行就行了。这个方法很方便，因为有时候一输 `claude` 咔嚓一下全清了，我就有点想不起来要干啥了。用这个就是一个完整的东西，不割裂，直接就从那个语句开始执行了，没有让你等待再输，很方便。

接下来就是无头模式。如果你觉得命令行太复杂，你可以交给它很多别的任务。往常你要安装工具、记复杂的命令，现在你可以用它处理命令行了。你根本不需要记任何指令，顶多它要用的话你帮它装一下就行，不需要管具体细节，它都会帮你搞定。这才是最牛逼的，Cursor 行吗？Cursor 不行啊，这东西是全局的。你在 Linux 上连图形界面都没有，你怎么用 Cursor？还要创建一个远程连接？那扯淡呢。不如直接 `claude` 一句话，什么都知道了。有了这个功能，你就可以抛掉什么 `tree` 命令、`ll` 命令、隐藏文件、文件大小，都不用记了，直接一句话问它就行了。我只讲市面上那些人没跟你讲过的东西。

## 自定义提示词模板与指令

接下来是提示词模板。你总是会有一些重复的命令、重复的 prompt，它其实还能加参数，但我没想好例子就没演示。你看这个是在 `.claude` 文件夹下面的 `commands` 目录里，你就在这下面写命令名字，然后 `.md` 就可以了。输入 `claude` 进来，打斜杠，你看我这里已经有这个命令了，斜杠输入 `commit` 就可以执行我写的指导命令。

这个超级好用，主要是我的那些东西没法直接展示，有些敏感内容我不想让别人知道我在做什么，所以只能告诉大家方法。具体结合你自己的项目一思考，就会发现这东西太牛逼了。重复的 prompt 可以弄成提示词模板、弄成指令，还可以传参数。具体的在 Claude 官网上搜一下就知道了，虽然是英语的，但可以用带翻译的浏览器看一下，很简单的。

## 子代理与并发执行：降维打击 Cursor

接下来这个我觉得是吊打 Cursor 的一个重要功能：它可以有子代理（Sub-agent）。Cursor 其实是串行的，一下一下问，不能递归。但这东西能递归！直接给大家看我之前写好的提示词，它可以同时搜索很多东西，同时分析很多文件，并发执行。Cursor 行吗？不行的。速度快 100 倍，尤其是项目特别复杂的时候，Cursor 动一动就几分钟，搜来搜去结果只能读 200 行，也写不出来啥。但这个是真的行。

你看它有基于 Codebase 的搜索，这东西其实 Claude Code 早有了，比 Cursor 早两三个月。我最早用就是因为它搜项目文件很方便，结果越做越牛逼，Cursor 反正也把这个抄过来了，这部分算打平手吧。一般就选第二个选项，让它一直写就行了，写得都挺准的。而且这个还能切换模型，它有一个 Claude Opus 模型超级强大。Claude 这个模型就是以前搞 OpenAI 那帮人弄出来的，所以一直都是地球上最好的模型。你得跟着最聪明的那帮创始人走，看他们在做什么，做出来的东西才是最牛逼的。听我的，就用 Claude Code，对 Cursor 是降维打击。

你看最开始用子代理分析了，只要有 Task，就是一个子任务。这里在分析代码结构，Cursor 是把所有文件全加到上下文里让大模型直接分析，比如你只能看 32 页的文章，相当于把 32 页全加进去，没有提取有用信息。子任务是什么意思呢？我知道这个任务需要占用很大的 Token，我就新建一个对话，把这 32 页全分析一遍，分析完之后把结果返回给父代理。这样父代理只有真正需要的东西，而不是子代理处理的那些杂事。这就是为什么它比 Cursor 聪明。而且它是并发执行的，搜索可以并发，嗖嗖三个一起搜索，快极了。

## 上下文记忆管理技巧

接下来讲一下记忆（Memory）的事。在你记忆长度快满的时候，它会在提示框右下角提示自动压缩。你不要等自动压缩，因为不知道它会提取出什么东西。你直接整理收拾代码，提交个 Git，然后重新开一个 Chat 就行了。这个其实是提醒你提交代码的，千万不要认为记忆长度是无限的，一定要让记忆里边是精华，而不是让它瞎压缩，那样信息密度不行的。
