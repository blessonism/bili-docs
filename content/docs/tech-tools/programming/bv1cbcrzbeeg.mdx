## 公共 WiFi 能看到你访问了哪些网站（但看不到具体内容）

计算机学生觉得是常识，但普通人不一定知道的事情。

当你连接了星巴克 WiFi 或者校园网 WiFi，开始刷手机的时候，你可能不知道这个 WiFi 的拥有者、管理员是可以看到你几点几分上了淘宝、几点几分上了哔哩哔哩、连了多久、用了多少流量这些信息的。

虽然他看不到你具体搜了什么、看了什么、买了什么（HTTPS 加密），但你去了哪些网站、待了多久，这些大都清清楚楚。

而且你开无痕模式也没有用，因为无痕模式是指不在你的设备上留下痕迹，路由器那边该记录的还是会记录。同理，删除浏览记录也没有用，因为你删除的是你手机设备上的，不是路由器那边的。

但是这里并没有说，在这个情况下可以看到任何具体内容。

## Cookie 到底是什么，你点“接受”同意了什么

很多人在浏览网页的时候都感到很烦，因为一开始都会弹出来一个问你接不接受所有的 cookies。很多人就直接点了“Accept all cookies”。

这个小曲奇到底是什么，你到底同意了什么？它其实就是网站存在你浏览器里的一个小文本文件。它会记录你浏览了哪些页面、点了什么链接、停留了多久，还有你的设备信息，比如浏览器类型、屏幕分辨率、操作系统，以及通过你的 IP 地址判断城市级别的位置信息（当然不会精确到门牌号）。

这些数据信息会被商家的数据团队用来分析用户行为，从而迭代他们的网站，促成更多的订单。

## 第三方 Cookie 的跨站追踪与“为什么我总看到同一双鞋的广告”

更可怕的是，一些第三方追踪 cookies 可以跨网站追踪你的浏览行为。这也就是为什么有时候你在某一个网站看了一双鞋，然后在一个完全不相干的网站又看到这双鞋的广告，这就是拜这些第三方 cookies 所赐。

## 免费 WiFi 的登录页（Captive Portal）与数据采集、用户画像

这块也和刚才说的那些免费 WiFi 有关系。那些免费的 WiFi 在让你使用之前，都会要求你登录，填入你的邮箱、手机号等个人信息，并且勾选一个“我已同意”（give consent）。

这块甚至有个专门的术语，叫 captive portal，就是强制门户认证页面。它的核心功能之一就是数据采集，然后建立用户画像，用来营销等。

现在市面上也有大量做 WiFi 营销的平台，他们等于是帮商家完成一个闭环：用户连接 WiFi，在登录页收集用户的邮箱等信息；用户开始使用 WiFi 浏览之后，就会把数据同步到他的 CRM 或其他管理工具之中。然后你每次来这个咖啡厅连这个 WiFi，这个系统都会持续构建你的用户画像，包括你的访问频率、还有一些偏好等。
