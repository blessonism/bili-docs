## Git 的基本概念与用途

Git 被写代码的人广泛使用。它不是一种编程语言，而是一个版本控制系统。它可以让程序员提交他们的代码贡献，而不用担心出错会带来严重后果。比如代码里有 bug，或者文件被不小心删除了。它还能帮你轻松查看谁在什么时候修改了什么。如果有人加了糟糕的代码，Git 允许你将这些修改回退到之前的快照，还能查出是谁做的，这样你就知道该怪谁了。

## 代码仓库与团队协作

Git 允许将代码和其他项目文件的集合（称为仓库）公开在 GitHub 和 GitLab 等托管网站上。在那里，它们可以被分享，甚至进行多人共同编辑。这极大地支持了团队协作。

## 在 JetBrains IDE 中使用 Git

接下来我们会使用 IntelliJ IDEA 进行演示，但它的操作界面和 WebStorm、PyCharm 或其他 JetBrains IDE 是一样的。首先，我们需要准备一个代码仓库。在 GitHub 上，如果我们要把仓库克隆下来，你可以使用命令行，但最简单的方法是直接使用 IDE 内置的功能来克隆仓库。

## 推送与拉取代码

从一个版本到下一个版本，一旦你确信你的代码没有 bug 并且经过了测试，你就可以准备推送（Push）你的修改了，这样你的其他协作者就能看到你的工作成果。但是等等，在你推送之前，仓库可能在此期间已经被其他人修改了。这意味着在推送你的修改之前，你需要先拉取（Pull）仓库的最新版本，这通常可以帮你解决代码冲突等问题。
